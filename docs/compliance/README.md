# docs/compliance

Store-submission and legal/privacy paperwork for Sonus Auris. Because the app
records audio continuously and requests sensitive permissions, both stores scrutinise
it heavily; these documents are the canonical source for the answers, kept in the
repo so they track the code that justifies them.

- **[PRIVACY_POLICY.md](PRIVACY_POLICY.md)** — the published privacy policy.
- **[PERMISSIONS_RATIONALE.md](PERMISSIONS_RATIONALE.md)** — why each runtime
  permission and sensitive API is requested (the reviewer justification).
- **[DATA_SAFETY_play.md](DATA_SAFETY_play.md)** — Google Play Data safety form
  answers.
- **[PRIVACY_LABELS_appstore.md](PRIVACY_LABELS_appstore.md)** — Apple App Store
  privacy "nutrition label" answers.
- **[EXPORT_COMPLIANCE.md](EXPORT_COMPLIANCE.md)** — iOS encryption export
  compliance (the app ships crypto; this documents the exemption/answers).
- **[ACCOUNT_DELETION.md](ACCOUNT_DELETION.md)** — the account/data deletion flow
  both stores require.
